### 謎の順序を作らない「one-hot表現」

[nanj:05] 彡(ﾟ)(ﾟ)「まずはラベルの表現からや。二値分類ではラベルは『0』か『1』の1つの値で済んだけど、多値分類ではどう表すのがええと思う？」

[yaruo:06] （ ＾ω＾）「単純に考えると、クラス0、クラス1、…、クラス4 って、背番号みたいに数字で表せばいいんじゃないかお？」

[nanj:07] 彡(-)(-)「それを『数値ラベル』って呼ぶんやが、これには致命的な問題がある。クラス0とクラス4は数字的に『4』も離れてるように見えるが、本来クラス間に大小関係や順序はないやろ？ 中心と右の点群に『どっちが偉い』とか無いんや」

[yaruo:08] （；＾ω＾）「あっ！ AIが勝手に『クラス4はクラス0の4倍の価値があるお！』って誤解しちゃうかもしれないんだお！」

[nanj:09] 彡(ﾟ)(ﾟ)「その通りや。そこで使うのが『one-hot表現』や。クラス数と同じ長さの配列（ベクトル）を用意して、該当するクラスの位置だけ『1』、それ以外は『0』にするんや」

[nanj:10] 彡(ﾟ)(ﾟ)「例えば5クラスならこうなる」

・クラス0 → `(1, 0, 0, 0, 0)`
・クラス1 → `(0, 1, 0, 0, 0)`
・クラス2 → `(0, 0, 1, 0, 0)`
・クラス3 → `(0, 0, 0, 1, 0)`
・クラス4 → `(0, 0, 0, 0, 1)`

[yaruo:11] （ ＾ω＾）「なるほどお！ 『1つだけホット（1）で、他はコールド（0）』だから one-hot表現って言うんだお。全員が等距離になって、贔屓（ひいき）がなくなるお！」

[nanj:12] 彡(^)(^)「せや。これで『クラス0とクラス4の距離』と『クラス0とクラス1の距離』が数学的に全く同じになって、全クラスが完全に対等に扱われるようになるんや」

### 確率の合計を1にする魔法「softmax」

[nanj:13] 彡(ﾟ)(ﾟ)「次は出力層や。二値分類ではsigmoidで『Bである確率 p』を1つだけ出した。Aである確率は自動的に『1-p』になったからな」

[nanj:14] 彡(ﾟ)(ﾟ)「でも多値分類では、各クラスに対する確率を全部出したい。だから出力層のノードを一気に5つに増やして、それぞれが『クラスiである確率 pi』を出す形にするで」

> [図2：計算グラフ（入力2 → 隠れ層 → 出力層5ノード）]

[nanj:15] 彡(-)(-)「ただし制約がある。確率なんやから、5つの `pi` は全て『0〜1の範囲』に収まってて、かつ『5人全員の確率を足したらピッタリ 1（100%）』にならなあかん。出力層の各ノードに単純にsigmoidをかけただけじゃ、この制約は満たせんのや」

[yaruo:16] （；＾ω＾）「確かにそうだお！ 各々が勝手に『俺は80%！』『ワイも70%！』って言い出したら、合計が300%とかになって、確率の概念が崩壊するお！」

[nanj:17] 彡(^)(^)「そこで登場するのが『softmax（ソフトマックス）関数』や！ 出力層の各ノードが吐き出した生の値（スコア）を `z1, z2, …, z5` とすると、softmaxは次のように計算するで」

`pi = e^(zi) / Σ_j e^(zj)`

[nanj:18] 彡(ﾟ)(ﾟ)「分子は各ノードの生の値を『指数関数 e^z』に通したもの。分母は『5人全員の e^z の総和』や。これで全体を割り算（正規化）してやるんや」

> [図3：softmax の計算イメージ図（5つの生の値が、合計1になるように5つの確率に変換される様子）]

[yaruo:19] （ ＾ω＾）「全体で割るから、絶対に合計が1になるんだお！ でも、なんでわざわざ `e^z`（指数関数）なんていう面倒な計算を挟むんだお？ そのまま足して割っちゃダメなのかお？」

[nanj:20] 彡(ﾟ)(ﾟ)「指数関数を使う理由は、『一番デカい値をより強烈に強調して、小さい値をほぼ無視する（ソフトな最大値）』ためや。これによって、一番可能性の高いクラスの確率がガツンと際立つようになるんやで。アイドルオーディションで一番人気のセンター候補に票が集中するイメージやな」

[nanj:21] 彡(^)(^)「ちなみに、softmaxはsigmoidの自然な正統進化系や。2クラスの時にsoftmaxの計算をすると、綺麗にsigmoidの式と一致するんやで（数学の神秘や）」

### 多クラス版の交差エントロピー

[nanj:22] 彡(ﾟ)(ﾟ)「次はLoss（損失）の定義や。第6話で『交差エントロピー』を導入したよな。多値分類では、これを多クラス版に拡張するで」

[nanj:23] 彡(ﾟ)(ﾟ)「正解のone-hotベクトルを `y = (y1, …, y5)`、さっきsoftmaxで出した予測確率を `p = (p1, …, p5)` とすると、式はこうなる」

`L = -Σ_i yi * log(pi)`

[nanj:24] 彡(^)(^)「ゴツい式に見えるが、ビビる必要はない。`y` はone-hotベクトルやから、正解クラスの位置だけ『1』で、他は全部『0』やろ？ つまり掛け算すると、正解以外の部分は全部消滅するんや。実質的には `L = -log(p_正解クラス)` っていう、超絶シンプルな形になるんやで！」

[yaruo:25] （ ＾ω＾）「おぉっ！ 結局『正解クラスの確率に対する -log』を計算しているだけだお！ 第6話の交差エントロピーと本質は全く同じだお！」

[nanj:26] 彡(ﾟ)(ﾟ)「その通りや。正解クラスの確率 `p` が 1 に近ければLossは0に近づき、`p` が 0 に近づけばLossは無限大に発散する。第6話で言った『知ったかぶりを絶対に許さない（自信を持って間違えると爆死する）』性質も健在やで」
